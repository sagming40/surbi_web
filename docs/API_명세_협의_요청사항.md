## 1. 문서 목적

Frontend 화면 개발 과정에서 "이런 데이터가 추가로 있으면 화면이 더 풍성해지겠다" 또는 "이 부분은 확정이 안 되면 다음 작업이 막힌다"고 판단된 항목들을 화면(Step)별로 정리한 문서.
API 명세 확정 시 BE(최민수)에게 전달할 요청 목록.

**보는 법**: 2번 요약표에서 항목을 먼저 확인 → 바로 답 가능하면 답변, 맥락이 더 필요하면 표의 "상세 섹션" 링크를 따라 아래로 이동.

## 2-1. 참고 — 이미 알고 있는 기존 제약사항

- `lat`, `lng`는 DB에서 nullable(좌표 없는 업소 존재) → 프론트 `Building` 모델은 현재 non-nullable(`double`)로 되어있어 API 연동 시 null 처리 로직 추가 필요 (Task 4-5 예정)
- `government_policy.dart`의 `pblanc_id`, `agency`/`jrsd_instt_nm` 필드명 불일치 건은 이미 확정 완료됨 (Task 3-6 반영 완료)

## 3. ⚠️ 우선 확인 필요 — buildingName 필드 매핑 이슈

`lib/models/building.dart`의 `buildingName` 필드가 DB의 무엇과 매핑되는지 아직 불명확함.

- DB `businesses.name` → 상호명 (예: "스타벅스 강남점")
- DB `businesses.building_name` → 건물명 (예: "강남파이낸스센터")

Step 3 기획 의도(BottomSheet에 "건물명" 표시)로 보면 `building_name` 쪽이 맞을 가능성이 높으나, 확정 전까지는 추측일 뿐.

## 4. Step 1 [지역·카테고리 선택] — 요청 필드

**전부 "있으면 좋은" 수준으로 우선순위는 낮음**

| 요청 필드 | 현재 상태 | 용도 | 비고 |
| --- | --- | --- | --- |
| 즐겨찾기(스크랩)한 지역 목록 | `favorites` 테이블 존재 확인(district_id 기준) | 검색창 위/아래에 "즐겨찾기" 칩 표시 | `favorites`가 지역 자체도 대상인지, 정책 전용인지 확인 필요 |
| 최근 조회한 지역 이력 | 없음(신규 요청) | 검색창 클릭 시 "최근 조회" 드롭다운 | 서버 저장 필요한지, 로컬(Firestore) 저장으로 충분한지 확인 |
| 지역별 인기 카테고리 Top N | 없음(신규 요청) | "망원동 인기 업종은 카페" 같은 추천 문구 | 백엔드 집계 vs 프론트 계산 여부 확인 |
| 자동완성 목록에 상권변화지표 배지 | `market_trends.trend_grade` 존재 확인 | 검색 후보에 "🔥 성장중" 배지 | Step 1 노출 여부 재검토 필요 |

## 7. Step 4 [AI 창업 점수 + LLM 보고서] — 요청 필드

`GET /areas/{id}/score` 응답 기준. DB `scores` 테이블은 `total_score`, `population_score`, `sales_score`, `age_target_score`, `competitor_score`, `ml_sales_score`, `transport_score`, `rent_score`, `model_version`, `score_reason` 총 10개 컬럼.

### 7-1. 종합 창업 점수 카드 (게이지)

| 요청 필드 | DB 컬럼(추정) | 용도 | 비고 |
| --- | --- | --- | --- |
| 종합 점수 | `total_score` | 게이지 숫자 | ✅ 매핑 완료 |

### 7-2. 예상 성과 카드 (예상 월매출 / 폐업 위험도)

## 11. 참고 — 시스템 문서 불일치 (별도 확인 권장)

> API 요청사항은 아니지만, 문서 정리 과정에서 발견된 항목이라 함께 기록해둠.

시스템 다이어그램(Mermaid/draw.io)상 **"Step 3 = 정부 지원사업 매칭"**으로 표기돼 있으나, 실제 구현 및 워크플로우 문서 기준으로는 **Step 3 = 건물 탐색(지도 모드)**이고, 정부 지원사업은 **Step 4 부속(4-3)**으로 구현됨. 다이어그램 최신화가 필요할 수 있어 최민수에게 별도 안내 권장.
